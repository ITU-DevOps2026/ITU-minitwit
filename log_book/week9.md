# Docker Swarm
In order to set up Docker Swarm, we first had to make sure that the "latest" value is persisted in the database instead of only in the application itself. This has to be done in order to make the Minitwit application stateless, so that it can be replicated (scaled horizontally) and all replicas will be able to see the same "latest" value. 

## Persisting Latest value in the database
We added another DbSet in our DbContext which is used to store the integer from the "latest" value. Based on this, we generated EF Core migrations, which we then converted to SQL using the `dotnet ef migrations script FollowerKey Persist_latest_value`. We then had to convert this SQL to use MySQL syntax instead. We then logged into the droplet for the database for the test environment and wrote the following command: `docker exec -it minitwit_mysql mysql -u root -p` followed by the password to the database. We could then write SQL queries to the database by first writing `USE minitwit` and then writing the command from the migrations that creates a new table: `START TRANSACTION; CREATE TABLE latest (id INT NOT NULL AUTO_INCREMENT, value INT NOT NULL, PRIMARY KEY (id)); COMMIT;`. We then checked that the new table was correctly created by running: `SHOW TABLES;`. After creating a pull request for our code changes, the application was automatically deployed to our test environment. We then ran a small simulator against our test application: `python minitwit_simulator.py "http://164.90.211.91:5035/api"` and ran `curl http://164.90.211.91:5035/api/latest` to verify that the latest value had been correctly persisted in the database. 

We then repeated the above process for our production application.
